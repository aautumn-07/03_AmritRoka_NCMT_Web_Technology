CHAPTER 2: WEBSITE DEVELOPMENT PROCESS

Group A: Short Questions
1. Define the term “Sitemap” in the planning phase.
=> A sitemap is a structured list or diagram of all web pages of a website. It helps in planning the website structure and navigation.

2. What is the purpose of the Launch phase in web development?
=> The launch phase makes the website live on the Internet so that real users can access and use it.

3. Why is documentation important in the development lifecycle?
=> Documentation records all technical and design details of a website. It helps future developers understand, maintain, and update the website easily.

4. Differentiate between Front-end and Back-end development.
=> Front-end development focuses on the design and user interface, while back-end development handles the server, database, and application logic.

Group B: Long Questions
5. Explain why planning is the most important phase of web development.
=> Planning defines website goals, target users, content, and features. It helps create sitemaps and wireframes, manage budget and time, and reduce errors, making development smooth and efficient.

6. Explain the iterative nature of the maintenance phase.
=> The maintenance phase continues after the website is launched. It includes updating content, fixing bugs, improving security, and making changes based on user feedback, which is why a website is never truly finished.

7. Describe the process of information gathering in web development.
=> Information gathering involves collecting client requirements, understanding business goals, identifying target users, and deciding content and features to ensure the website meets user needs.

Group C: Scenario-Based Questions
8. Explain the risks of skipping the testing phase.
=> Skipping testing can lead to security issues, broken features, and compatibility problems. Errors after launch reduce user trust and damage the website’s reputation.

9. Analyze what went wrong when a developer started coding without planning.
=> The developer skipped planning and design stages such as sitemap and wireframe creation. Client expectations were unclear, leading to layout rejection and wasted time and cost.

10. Which phase was neglected when website content became outdated? Explain.
=> The maintenance phase was neglected because content was not updated regularly. This phase includes updating information, fixing issues, and keeping the website useful for users.
